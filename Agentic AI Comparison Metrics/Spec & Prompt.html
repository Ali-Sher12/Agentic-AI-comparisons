Master Prompt (paste verbatim prompt used for all 3 tools)

7
Build a full-stack web app for a Lost & Found system used by police across Pakistan. The app runs entirely locally on my PC — no hosting, no domain, no cloud services.
8
9
Project structure: single repo with two folders: /frontend and /backend.
10
11
Stack:
12
13
Frontend: React + Vite + Tailwind CSS, running on localhost:5173
14
Backend: Node.js + Express, running on localhost:3001
15
Database: SQLite via Prisma ORM — single file at backend/prisma/dev.db
16
Config/secrets (JWT secret, DB url, etc.) in a .env file in /backend, with a .env.example provided
17
File uploads (claim documents): stored locally in backend/uploads, file path saved in the database
18
19
Seed data: include a Prisma seed script that creates 4 sample police HQs (e.g., Lahore HQ, Karachi HQ, Islamabad HQ, Peshawar HQ), one police login per HQ, and 5–6 sample lost items with varied categories, so the app is testable immediately after setup without manual data entry.
20
21
Police side:
22
23
Login required, one account per HQ/building
24
All HQs can see and manage items logged by every other HQ (shared visibility, not siloed)
25
"Add Item" form with fields: size, weight, color, detailed description, number plate (only relevant if item is a transport/vehicle), condition found in, recovered-from location, recovery time and place, holding location (which HQ/building currently has it)
26
Upload date is set automatically on creation, not user-entered
27
Police can mark an item as "Returned to Owner"
28
Returned items are never deleted — archived for historical/audit purposes
29
Return status is irreversible once set
30
When marking returned, police record who it was returned to — visible only to police, never to the public
31
Police can view incoming claims for their items and Accept or Reject each one
32
33
Civilian side:
34
35
No login or account required
36
Can search and filter items by all fields above
37
Can submit a claim on an item: email or CNIC, contact info, uploaded proof-of-ownership document
38
Maximum 3 claims per person (by email/CNIC) per item
39
Rejected claims still count toward that 3-claim limit
40
Civilians cannot see who returned items were given to
41
42
Setup requirements:
43
44
Must run with simple local commands only: npm install (in both /frontend and /backend), npx prisma migrate dev, npx prisma db seed, npm run dev
45
No Docker, no cloud database, no external services
46
Uninstalling = deleting the project folder
